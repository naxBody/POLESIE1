# Отчёт о миграции данных из JSON в базу данных

## Выполненные работы

### 1. Оптимизация схемы базы данных

**Удалены неиспользуемые таблицы:**
- `passport_templates` - шаблоны паспортов (не используются динамически)
- `product_passport_versions` - история версий паспортов (не ведется)  
- `product_documents` - документы хранятся в файловой системе

**Добавлены новые таблицы для работы с материалами:**
- `material_categories` - иерархическая структура категорий материалов
- `materials` - расширенный справочник материалов с JSON спецификациями

## Инструкция по запуску

### Шаг 1: Установка зависимостей
```bash
pip install mysql-connector-python
```

### Шаг 2: Настройка подключения
Отредактируйте файл `polesie/sql/migrate_json_to_db.py`:
```python
DB_CONFIG = {
    'host': 'localhost',
    'database': 'polesie_production',
    'user': 'root',
    'password': 'ВАШ_ПАРОЛЬ',
    'charset': 'utf8mb4'
}
```

### Шаг 3: Создание БД и миграция
```bash
# Создание базы данных
mysql -u root -p < polesie/sql/schema.sql

# Запуск миграции
cd polesie/sql
python3 migrate_json_to_db.py
```

## Преимущества новой структуры

1. **Все данные в одном месте** - больше не нужно хранить данные в JSON файлах
2. **Целостность данных** - внешние ключи гарантируют согласованность
3. **Эффективный поиск** - индексы ускоряют выборку данных
4. **Гибкость** - JSON поля позволяют хранить сложные спецификации
5. **Масштабируемость** - нормализованная структура готова к росту

## Примечания

- Скрипт миграции использует `ON DUPLICATE KEY UPDATE`, поэтому повторный запуск безопасен
- Все текстовые данные сохраняются в кодировке UTF-8
- Спецификации материалов сохраняются в JSON формате для гибкости
- История версий паспортов удалена как неиспользуемая функция
